AlayaMarket Offers External Events APIÂ 1.0.0
  • application/json

Async API for external Offer events using AWS SNS as a broker.

Generally, subscriptions are expected to be https endpoints. In such cases, the messages described in this document will be stringified json data within the Message of the payload.

For more information see https://docs.aws.amazon.com/sns/latest/dg/SendMessageToHttp.prepare.html

Operations

  • SUB outbox-offers

    Subscribe demand participants to outbox offer events published by the service.

    Operation IDsub_outbox_offers

    Accepts the following message:

    Outbox offer state change eventmessage_outbox_offer

    Inform relevant demand participants about a change to the offer state

    Message IDmessage_outbox_offer
    object
    uid: OutboxOfferEventPayload
    event_source
    string

    Source of the event.

    Const:"alayamarket.offers"
    event_name
    string

    Name of the event.

      Allowed values:
    • "OfferAccepted"
    • "OfferDeclined"
    id
    string
    format: uuid

    ID of the offer

    Additional properties are allowed.

    Examples

    {
  "event_source": "alayamarket.offers",
  "event_name": "OfferAccepted",
  "id": "497f6eca-6276-4993-bfeb-53cbbbba6f08"
}

  • SUB outbox-visit-tasks

    Subscribe demand participants to outbox visit task events published by the service. This is only supported for offers of type 'visit'.

    Operation IDsub_outbox_visit_tasks

    Accepts the following message:

    Outbox visit task state change eventmessage_outbox_visit_task

    Inform relevant demand participants about a change to one (or more) visit task state

    Message IDmessage_outbox_visit_task
    object
    uid: OutboxVisitTaskEventPayload
    event_source
    string

    Source of the event.

    Const:"alayamarket.visit_tasks"
    event_name
    string

    Name of the event.

      Allowed values:
    • "VisitTasksCompleted"
    id
    string
    format: uuid

    ID of the visit.

    visit_outbox_id
    string

    ID of the visit in the demand system associated with the visit.

    array<object>
    id
    string
    format: uuid

    ID of the visit task.

    outbox_id
    string

    ID of the visit task in the demand system.

    is_completed
    boolean

    Is the visit task completed?

    completed_at
    string
    format: date-time

    The date-time at which the visit task was completed.

    completion_comment
    string

    Comment related to the completion of the visit task.

    Additional properties are allowed.

    Examples

    {
  "event_source": "alayamarket.visit_tasks",
  "event_name": "VisitTasksCompleted",
  "id": "497f6eca-6276-4993-bfeb-53cbbbba6f08",
  "visit_outbox_id": "string",
  "tasks": [
    {
      "id": "497f6eca-6276-4993-bfeb-53cbbbba6f08",
      "outbox_id": "string",
      "is_completed": true,
      "completed_at": "2019-08-24T14:15:22Z",
      "completion_comment": "string"
    }
  ]
}
